7.4.2. Expresiones

¿Cuáles son las expresiones que podemos utilizar en una expresión XPath?

  • Paréntesis, “()”; llaves , “{}” y corchetes, “[ ]”.
  • Atributo, “@”.
  • El nombre de un elemento
  • Tipo de nodo. Puede haber varios tipos pero el que representa el contenido textual de un elemento o atributo, text() es el más utilizado.

Ejemplo con corchetes "[ ]"

Si se sabe que una expresión devolverá varios resultados pero sólo se quiere uno específico se puede usar un número rodeado por corchetes cuadrados "[]" para indicar qué es lo que se quiere conseguir. Para devolver sólo el primer alumno puede hacer lo siguiente:

/clase/alumnos/alumno[1]

Solución

Devolverá:

< alumno>
< nombre>Felipe< /nombre>
< apellidos>Aranburu< /apellidos>
< /alumno>

Se pueden usar los corchetes en cualquier lugar de la expresión para hacer determinar cuál de las ramas se elegirá. Por ejemplo, se puede obtener sólo el segundo alumno con una expresión como esta:

/clase/alumnos/alumno[2]/nombre

Ejemplo atributos

Los valores de los atributos se pueden conseguir especificando el símbolo @ delante del nombre una vez se haya alcanzado el elemento que lo contiene:

/clase/profesor/@especialidad

Solución

Hay que tener en cuenta que a diferencia de lo que ocurre con los elementos, al obtener un atributo no tendremos un elemento sino sólo su valor:

especialidad="507"

Ejemplo text()

Para cuando sólo queramos el contenido de un elemento, podemos utilizar la función text() para obtener el contenido:

/clase/profesor/nombre/text()

Solución

... devolverá el contenido del nodo sin las etiquetas.

Otra expresiones que también nos podemos encontrar:

  • Elemento, “*”. Por ejemplo, si queremos obtener todos los elementos dentro del nodo profesor:
/clase/profesor/*
  • Las dobles barras "/ /" indican que cuadrará con cualquier cosa desde el nodo en el que estamos. Puede ser un solo elemento o un árbol de nodos. Por ejemplo, si queremos obtener todos los elementos nombre del archivo independientemente del lugar donde estén dentro del documento XML:
//nombre


    

  • Se pueden poner las dobles barras en cualquier lugar dentro de la expresión para indicar que puede haber cualquier cosa en medio a partir del lugar donde aparezcan. Aunque facilita la creación de expresiones no es muy recomendable abusar del comodín // por motivos de eficiencia.Las expresiones con este comodín requerirán muchos más cálculos para ser evaluadas, y por lo tanto las expresiones tardarán más en dar resultados. 
    • 





/clase/alumnos//nombre




    

  • Operadores: and, or, mod, div, *, /, //, |, +, -, =, !=, <, >, <=, >=.
    • 

  • Separador “::”.
    • 

  • Elemento actual "." y elemento padre ".."
    • 

  • Coma, “,”.
    • 

  • Ejes, son una serie de elementos que permiten hacer referencia a partes del árbol:

      

    • child ::, selecciona el hijo del elemento actual. Su forma habitual es la barra, “/”, aunque también puede ponerse: /child::
      • 

    • attribute::, permite seleccionar los atributos del elemento actual.
      • 

    • descendant::, permite seleccionar todos los nodos que descienden del nodo actual. Se corresponde con la doble barra, //, aunque se puede usar: descendant::
      • 

    • self::, se refiere al nodo contexto y se corresponde con el punto ”.”.
      • 

    • parent::, selecciona los nodos padre, para referirnos a él usamos los dos puntos, “..”.
      • 

    • 

      Por ejemplo la siguiente expresión:
      

      

      /clase/profesor/nombre
      

      

      la podríamos escribir utilizando ejes como:
      

      

      /child::clase/child::profesor/child::nombre
      

      

      • 

    

    • 

  • Nombres de función. (los veremos en el punto 1.5)
    • 

  • Literales, se ponen entre comillas dobles o simples. Pueden anidarse alternando el tipo de comillas.
    • 

  • Números.
    • 

  • Referencias a variables, para lo que se utiliza la sintaxis: $nombreVariable
